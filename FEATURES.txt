1. AI COLLABORATION FOUNDATION
======================================================================

Purpose:

Provide a structured foundation for long-term collaboration between
the User and AI Assistant.

Key Capabilities:
- Defined User authority.
- Defined Assistant behavior boundaries.
- Defined output responsibilities.
- Defined documentation rules.
- Reusable working context.

Benefits:
- Reduces ambiguity.
- Improves continuity.
- Makes AI-assisted work easier to review and maintain.

======================================================================
2. CONTROLLED INITIALIZATION
======================================================================

Purpose:

Establish a known package context before active work begins.

Key Capabilities:
- Package asset discovery.
- Required asset validation.
- Controlled initialization.
- Applicable asset activation.
- Exact ready-state response.
- Optional asset activation when authorized or required.

Benefits:
- Reduces repeated setup.
- Establishes a validated operational foundation.
- Clearly separates required and optional package assets.

Boundary:

Initialization does not guarantee identical behavior across all AI
systems.

Actual behavior remains subject to model capability, platform support,
available tools, context limits, and active instructions.

Successful initialization activates the Core Assets.

Optional Assets are activated only when required by the current task,
explicitly selected by the User, or authorized by an applicable
workflow.

- Clear User and Assistant responsibility boundaries.
- Standards-driven documentation workflows.

limitations, or replace professional judgment and external validation.
